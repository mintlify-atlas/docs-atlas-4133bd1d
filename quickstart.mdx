---
title: "Alice quickstart: write and load your first plugin"
description: "Create and load your first Alice plugin in minutes. This guide walks through writing manifest.json, extending PluginBase, and enabling your plugin."
---

This guide takes you from a fresh clone of Alice to a running plugin that registers a UI tab and listens to core events. You'll write two files — `manifest.json` and `plugin.py` — and see them loaded by the Alice runtime on startup.

<Steps>
  <Step title="Set up your environment">
    Clone the Alice repository and install its dependencies:

    ```bash
    git clone https://github.com/runesc/Alice
    cd Alice
    python -m venv .venv
    source .venv/bin/activate      # Windows: .venv\Scripts\activate
    pip install -r requirements/base.txt
    ```

    For platform-specific packages, install the file that matches your OS:

    ```bash
    # Linux
    pip install -r requirements/linux.txt

    # macOS
    pip install -r requirements/mac.txt

    # Windows
    pip install -r requirements/windows.txt
    ```
  </Step>

  <Step title="Create your plugin directory">
    Alice discovers plugins by scanning for subdirectories inside `plugins/`. Each plugin lives in its own folder named after its reverse-domain ID.

    ```
    plugins/
    └── com.example.hello/
        ├── manifest.json
        └── plugin.py
    ```

    Create the directory:

    ```bash
    mkdir -p plugins/com.example.hello
    ```

    <Note>
      The `plugins/` directory sits at the root of the Alice repository, next to `src/`. Alice creates it automatically on first run if it does not exist.
    </Note>
  </Step>

  <Step title="Write manifest.json">
    The manifest is the contract between your plugin and the Alice runtime. Every field is validated before any plugin code runs. Create `plugins/com.example.hello/manifest.json`:

    ```json
    {
      "schema_version": "1",
      "id": "com.example.hello",
      "name": "Hello Plugin",
      "version": "1.0.0",
      "entrypoint": "plugin:HelloPlugin",
      "permissions": [
        "ui.tab",
        "storage.read",
        "storage.write",
        "events.subscribe",
        "events.emit"
      ],
      "api_compatibility": {
        "min": "1.0.0",
        "max": "1.x"
      },
      "dependencies": {
        "python": [],
        "plugins": []
      }
    }
    ```

    Key fields:

    - **`id`** — must be a reverse-domain string matching the pattern `^[a-z0-9]+(\.[a-z0-9\-]+)+$`.
    - **`entrypoint`** — format is `module:ClassName`. Alice imports `plugin.py` from your plugin directory and instantiates `HelloPlugin`.
    - **`permissions`** — every `self.context.*` API call is checked against this list. Undeclared permissions raise a `PermissionError` at runtime.
    - **`api_compatibility`** — Alice checks this range against its own API version before loading.
  </Step>

  <Step title="Write the plugin class">
    Create `plugins/com.example.hello/plugin.py`. Your class extends `PluginBase` and implements lifecycle hooks:

    ```python
    from core.skills.base import PluginBase


    class HelloPlugin(PluginBase):

        def on_load(self):
            self.context.logger.info("Hello plugin loaded")

        def on_enable(self):
            from PySide6.QtWidgets import QLabel

            self.context.ui.register_tab(
                lambda: QLabel("Hello from my plugin!"),
                title="Hello"
            )
            self.context.events.subscribe(
                "core.plugin.enabled",
                self._on_plugin_enabled
            )

        def _on_plugin_enabled(self, event: str, payload: dict):
            self.context.logger.info(f"Another plugin enabled: {payload}")

        def on_disable(self):
            self.context.logger.info("Hello plugin disabled")

        def get_main_widget(self):
            from PySide6.QtWidgets import QLabel
            return QLabel("Hello World!")
    ```

    Lifecycle summary:

    - `on_load` — called every startup while the plugin is enabled. Avoid heavy I/O; it must finish in under 500 ms.
    - `on_enable` — called when the user activates the plugin. Mount UI and subscribe to events here.
    - `on_disable` — called when the user deactivates the plugin. Alice automatically unsubscribes all event listeners registered under your plugin ID, so you only need to explicitly clean up other resources.

    <Tip>
      Always use `self.context.logger` instead of `print()`. Log output goes to `debug.log` in the project root and is prefixed with your plugin ID, making it easy to filter.
    </Tip>
  </Step>

  <Step title="Run Alice">
    With your plugin directory in place, start Alice:

    ```bash
    python src/main/python/main.py
    ```

    On startup, `PluginManager` scans `plugins/`, validates your manifest, and calls `on_load`. Enable your plugin from the settings panel in the Alice UI to trigger `on_enable` and see the "Hello" tab appear.

    Check `debug.log` in the project root for log output from your plugin.
  </Step>
</Steps>
