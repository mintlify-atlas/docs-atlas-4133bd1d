Alice is a desktop AI assistant framework designed to be extended through plugins. Every feature in Alice — from UI panels to background services — is delivered as a plugin that declares its permissions, dependencies, and lifecycle hooks in a `manifest.json` file. The core runtime handles discovery, loading, sandboxing, and inter-plugin communication so you can focus on building capabilities.

## How Alice works

Alice is built around a few core primitives that every plugin developer interacts with:

<Steps>
  <Step title="Declare a manifest">
    Each plugin ships a `manifest.json` describing its ID, version, entrypoint, required permissions, and API compatibility range. Alice validates this before loading anything.
  </Step>
  <Step title="Extend PluginBase">
    Your plugin class inherits from `PluginBase` and implements lifecycle hooks (`on_load`, `on_enable`, `on_disable`, etc.). Alice calls these at the right moments.
  </Step>
  <Step title="Use PluginContext">
    Alice injects a `PluginContext` into every plugin instance, giving access to the UI bridge, event bus, isolated storage, inter-plugin services, and a scoped logger — all permission-gated.
  </Step>
  <Step title="React to events">
    Plugins communicate with each other and with the core through the `EventBus`. Subscribe to system events, emit namespaced events, and build reactive features without tight coupling.
  </Step>
</Steps>
